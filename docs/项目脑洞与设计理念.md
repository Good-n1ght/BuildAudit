# 硬件对比助手 — 项目脑洞与设计理念存档

> 本文档存档项目诞生以来所有关键讨论、设计决策、脑洞演变过程。

## 1. 项目起源

- **初衷**：做一个本地硬件数据库 + 打分引擎 + 配置推荐，帮助小白装机避坑
- **第一个里程碑**：手工整理 hardware_db.json，覆盖 CPU/GPU/主板/内存/SSD/散热/电源/机箱 8 大件
- **最终形态**：不是一个静态表格，而是一套可进化、可搜索、可打分的硬件知识系统

## 2. 数据库扩建历程

### 2.1 内存 (v0.1.1)
- 从 184 条 → 230 条，覆盖 30 品牌
- 引入 T1/T2/T3 品牌梯队（金士顿/芝奇/海盗船 → 光威/阿斯加特 → 博帝/金邦等）
- 关键决策：涨价分界线定 2024 年，全部 25 品牌均成立于涨价前
- 新增 cl / height 字段，支撑后续打分

### 2.2 SSD (v0.1.2)
- 从 54 条 → 290 条，覆盖 30 品牌
- T1 六大原厂（三星/西部数据/致态/英睿达/SK海力士/铠侠）
- T2 八大模组厂（金士顿/威刚/雷克沙/海力士P41等）
- T3 十六国产新锐（达墨/梵想/爱国者/移速等）
- 新增 has_dram / nand_type / tbw 字段
- 关键纠正："西数"与"西部数据"合并去重
- QLC 产品标记、980 PRO 小容量无 DRAM 精确区分

### 2.3 电源 (v0.1.3)
- 从 82 条 → 245 条，覆盖 32 品牌
- T1 一线（海韵/振华/台达/全汉）
- T2 主流（长城/航嘉/华硕/海盗船/微星等）
- T3 性价比（酷冷/鑫谷/先马/骨伽等）
- 新增 atx_version / capacitor / fan_size / warranty
- 评判术语统一化（白金→白金牌）
- 大水牛等明确标记电子垃圾
- ATX 3.1: 41 条 / ATX 3.0: 69 条 / ATX 2.x: 98 条

### 2.4 散热器 (v0.1.4)
- 从 61 条 → 156 条，覆盖 25 品牌
- 风冷 34→62 / 水冷 27→94
- 风冷新增 tower(单塔/双塔/下压式) + fan_count
- 水冷新增 radiator_size(240/280/360/420) + has_screen
- 移除不通用字段（height 仅风冷保留）

### 2.5 GPU 补充

### 2.6 机箱 (v0.1.5)

## 3. 架构设计

### 3.1 五层架构
```
data  → engine → api → ui
  ↑                 ↓
  └── updater ←─────┘
```

| 层 | 职责 | 纪律 |
|---|---|---|
| **data** | 数据结构定义 + JSON 主库 + 迁移脚本 | 不写计算逻辑、不碰 UI |
| **engine** | 打分引擎 / 兼容性检查 / 搜索索引 | 不碰 UI，输出结构化数据 |
| **api** | 对外接口层，封装 engine 调用 | 前端唯一入口 |
| **ui** | 前端展示 | 通过 api 拿数据 |
| **updater** | 数据更新/校验/索引重建 | 不跳过 data→engine→api 调用链 |

### 3.2 核心纪律
- **数据层不写逻辑**：schema 定义字段，不写评分计算
- **引擎不碰 UI**：只输出 JSON 或结构化结果
- **前端走 API**：不直接读 JSON、不直接调 engine
- **自动重建索引**：数据更新后自动触发搜索索引重建

## 4. 搜索问题剖析

### 4.1 现状
- 裸 JSON substring 匹配，无索引
- CPU 搜 "9" 返回 35 条杂音（Intel 9xxx / Ryzen 9 / i9 全混一起）
- GPU 搜"电竞判客"命中 0 条（数据库用"叛"字）

### 4.2 解决方案
- SQLite FTS5 全文索引
- 支持模糊匹配、拼音匹配、别名映射
- 数据更新时自动触发索引重建
