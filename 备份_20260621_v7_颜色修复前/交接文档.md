# PC硬件搭配评分应用 — 项目交接文档

## 1. 项目概述

**项目名称**：PC硬件搭配评分应用（赛博跑分）

**核心功能**：用户选择CPU/主板/显卡后，系统基于8项兼容性规则和6维评分维度，生成"赛博跑分"并给出搭配建议。支持三种装机策略（性价比 / 一步到位 / 面向未来），帮助用户判断硬件搭配是否合理。

**当前版本**：v1.0 — 全品类2164款硬件覆盖

**技术栈**：Python 3（评分引擎） + JSON（数据库） + 纯HTML/JS（前端，零依赖）

---

## 2. 架构说明

| 组件 | 文件 | 职责 |
|------|------|------|
| 评分引擎 | `scoring_engine.py` | 纯规则计算：兼容性检查 + 6维评分 + 三种策略权重 |
| 硬件数据库 | `hardware_db.json` | 2164款硬件结构化数据，7大品类 |
| 前端应用 | `cyber_score.html` | 自包含HTML：搜索框 + 数据库embed + 评分逻辑JS版本 + 赛博跑分卡片 |

**设计原则**：前端HTML内嵌数据库并实现了评分引擎的JavaScript版本，可离线单文件使用。Python评分引擎用于命令行/批处理场景，两者逻辑完全一致。

## 3. 评分核心逻辑

### 3.1 8大件兼容性检查

| 检查项 | 函数 | 说明 |
|--------|------|------|
| CPU-主板接口 | `check_socket_compatibility` | socket必须一致，否则致命错误 |
| 内存代数 | `check_memory_compatibility` | CPU/主板共同支持的内存类型取交集 |
| 主板供电 | `check_vrm_for_cpu` | CPU TDP ≥170W + VRM评分≤4 → 警告；TDP ≥125W + VRM≤3 → 致命 |
| 电源功率 | `check_psu_capacity` | (CPU TDP + GPU TDP + 50W) / 电源功率 > 80% → 警告 |
| 机箱兼容 | (待实现) | 显卡长度/散热器高度/主板版型 vs 机箱限制 |

### 3.2 6维评分维度

| 维度 | 权重(性价比) | 权重(一步到位) | 权重(面向未来) | 说明 |
|------|:---:|:---:|:---:|------|
| CPU-GPU匹配 | 30% | 25% | 20% | 性能分级1-7级，同级别95分，差1级78分，差2级55分，差≥3级30分 |
| 供电合理性 | 20% | 30% | 20% | VRM相数×MOS电流评分，余量充足加分，供电偏紧扣分 |
| 散热余量 | 15% | 25% | 15% | 散热器TDP / CPU TDP比值：≥2.0倍95分，≥1.5倍82分，≥1.2倍65分，<1.0倍15分 |
| 升级空间 | 15% | 10% | 35% | AM5+30分，LGA1851+20分，PCIe5.0+10分，4内存槽+5分 |
| 电源余量 | 10% | 10% | 10% | 负载率≤50%得92分，≤70%得78分，≤80%得55分，>80%得25分 |
| 参数利用率 | 10% | 0% | 0% | 仅性价比策略启用：VRM利用率 + PCIe利用率 |

### 3.3 三种装机策略

- **性价比**：重点CPU-GPU匹配(30%) + 参数利用率(10%)，不浪费供电/散热资源
- **一步到位**：重点供电(30%) + 散热(25%)，确保当前配置稳定满载
- **面向未来**：重点升级空间(35%)，重视平台寿命和接口前瞻性

## 5. 别名系统说明

前端内置了 `ALIASES` 映射表（约2000+条别名），支持用户用俗称/简称搜索硬件：

| 别名类别 | 别名示例 | 映射目标 |
|----------|----------|----------|
| 主板俗称 | `迫击炮`, `重炮手`, `吹雪`, `小雕`, `雕妹` | 对应完整型号 |
| CPU简称 | `12400f`, `7500f`, `7800x3d`, `9800x3d` | 完整 CPU 名称 |
| 显卡简称 | `4060`, `4070s`, `4080s`, `9070xt` | 完整显卡名称 |
| 品牌+芯片组 | `华硕 b650`, `微星 x870` | 该品牌代表性型号 |
| 寨板/洋垃圾 | `华南 x99`, `x79`, `劲鲨 x99` | 对应完整型号 |
| 特殊板型 | `b650m-k`, `ayw`, `v20`, `v14` | 华硕/七彩虹特定型号 |

别名匹配机制：用户输入 → 去除空格小写 → 查 `ALIASES[category]` → 返回完整名称 → 再查 `DB[category]`。

## 6. HTML前端功能

### 6.1 搜索框

- 支持模糊搜索：输入任意部分产品名/别名/简称即可匹配
- 支持品牌过滤：输入品牌名缩小结果范围
- 实时结果展示：下拉列表显示匹配型号

### 6.2 结果展示

- 选定CPU/主板/显卡后，触发评分引擎（JS版本）
- 显示"赛博跑分"总分和进度条
- 六维评分雷达/条形图展示
- 标红项（严重的兼容性问题）红色高亮
- 加分项/降分项逐条列出
- 一句话点评

### 6.3 策略切换

- 三个按钮切换：性价比 / 一步到位 / 面向未来
- 切换后自动重新计算评分
- 各维度权重动态变化

### 6.4 自包含特性

- 数据库以 `const DB = {...}` 嵌入HTML
- 评分逻辑以纯JavaScript实现于 `<script>` 标签中
- 零外部依赖，双击即用

## 8. 后续开发建议

### 8.1 机箱反向需求卡逻辑

当前未实现机箱品类。建议新增 `case` 品类数据库，字段包含 `max_gpu_length`、`max_cooler_height`、`form_factor`，在检查中增加：
- 显卡长度 `gpu.length` vs `case.max_gpu_length`
- 散热器高度 `cooler.height` vs `case.max_cooler_height`
- 主板版型 `mb.form_factor` vs `case.form_factor`

### 8.2 洋垃圾标注规则

当前 `is_e_waste` 布尔值过于简单。建议细化：
- `e_waste_level`: `none` / `suspicious` / `confirmed` 三级
- 自动标注规则：LGA2011-3 / X79 / X99 寨板 + E5 V2/V3 组合 → `confirmed`
- 前端展示醒目的洋垃圾警告卡片

### 8.3 前端体验优化

- 搜索框支持拼音首字母匹配（如输入 `psj` → 匹配 `迫击炮`）
- 价格字段：加入 `price` 字段后启用价格/性价比维度
- 分享功能：生成配置方案链接/截图
- 对比模式：两个配置并排对比

### 8.4 数据库维护

- 定期补充新品（RTX 60系列、RX 10000系列发布时）
- 厂商型号去重：同一款主板常有多个SKU变体（WiFi版/非WiFi版/Revision版），需做好去重
- 批量生成脚本保留：显卡品类的批量生成逻辑可复用于未来新架构显卡

### 8.5 评分引擎

- 内存兼容性深化：当前仅检查代数，建议增加QVL清单校验
- 多GPU支持：双卡计算/NVLink场景
- ITX小机箱特化：对ITX版型自动增加体积/散热/兼容性权重
