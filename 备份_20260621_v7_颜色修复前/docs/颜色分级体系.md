# 装备颜色分级体系

> 评分后根据最终得分映射为颜色，用于前端醒目展示。  
> 更新于 2026-06-21，按用户最新指令调整。

## 颜色映射表

| 分值区间 | 颜色 | CSS 色值 | 含义 |
|---|---|---|---|
| 0 ~ 59 | **白色** | `#B0B0B0` | 不推荐 / 电子垃圾级别 |
| 60 ~ 69 | **绿色** | `#4CAF50` | 能用级别 |
| 70 ~ 79 | **蓝色** | `#2196F3` | 合理水平 |
| 80 ~ 89 | **紫色** | `#9C27B0` | 优秀配置 |
| 90 ~ 95 | **金色** | `#FFD700` | 顶级装备 |
| 96 ~ 99 | **红色** | `#F44336` | 极致拉满 — 所有硬件全部拉满，且配到合适场景+机箱 |
| 100 | **纯黑 / 暗金** | `#1A1A1A` / `#B8860B` | 至高无上 — 仅真正最顶级的 1~2 款，全硬件满分+场景完美+机箱完美 |

## 设计原则

1. **90~95 金色**：常规天花板，顶级但还未触达极限边界
2. **96~99 红色**：极限配置，不仅硬件拉满，场景和机箱也完美匹配
3. **100 至高无上**：真正的唯一，所有维度没有任何妥协空间，仅 1~2 款能够达成
4. **不到 100 不是失败**：99 已经是极限中的极限，100 是理论完美值

## 分数来源

最终得分由 `engine/scorer.py` 的计算结果决定：
- 硬件加权分（各配件 tier_score 加权）
- 兼容性加分（配件之间匹配度）
- 场景适配加分（机箱能否装下、散热是否够用）
- 扣分项（电子垃圾配件 / 明显瓶颈搭配）

## 附加映射：单品颜色

单品（CPU / GPU 等）也可映射颜色：

| 单品 tier_score | 颜色 |
|---|---|
| 1 ~ 3 | 白色 |
| 4 ~ 5 | 绿色 |
| 6 ~ 7 | 蓝色 |
| 8 | 紫色 |
| 9 | 金色 |
| 10 | 红色 |
